## 專案目標

建立一個包含 **997 個分層抽樣人設** 的資料庫，代表台灣 2,300 萬人口結構，供 **GEO（Generative Engine Optimization）Operator** 在回應各領域問題時，載入對應的人設視角以提升回答精準度。

## 核心機制

### 雙 Prompt 格式

每個 Persona 包含兩種 prompt：

1. **`prompt_prefix`** — 完整沉浸式角色代入，含背景故事 + 興趣 + 政治立場
2. **`reference_pre_prompt`** — 結構化摘要，供 GEO Operator 快速篩選比對

### 人口加權抽樣

```
真實 API 數據（ODRP014）→ 年齡×性別×區域人口權重
    → 加權分配到 112 種基礎組合（7區域 × 8年齡 × 2性別）
    → 其他維度依機率分配填入
    → 最終 997 人設
```

### 連貫性驗證

生成後自動檢查並修正反直覺組合：
- 5+ 人家庭不得低於 3-7萬 家庭收入
- 4 人家庭不得低於 1-3萬
- 未成年不得設定已婚
- 未成年不得設定政治傾向

### 背景故事推論

從維度組合自動推導合理背景（推論引擎內建 ~30 條規則）：
```
19-24 / 未婚 / 學生 / 2人家庭
→ 跟媽媽同住，是單親家庭 → 正在讀大學

19-24 / 未婚 / 學生 / 5+人家庭
→ 家裡排行老大，底下還有弟妹 → 有時需要打工貼補家用

35-44 / 已婚 / 4人家庭
→ 典型小家庭，夫妻加兩個孩子 → 上有老下有小，是家庭支柱
```

## 已知限制

| 限制 | 說明 |
|------|------|
| API 覆蓋率 | ODRP014 僅回傳前 2000 筆（3 縣市），其他區域用估計比例補足 |
| 職業分配 | 職業機率為合理估計，非來自真實統計 |
| 收入分配 | 個人收入與家庭收入皆為機率模型估計 |
| 名字快取 | 相同 age/sex/occ 組合會共用同一名字，導致名字重複率偏高 |

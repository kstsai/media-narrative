## 1. 起點：RFC 規格需求

根據 `tw-persona-db-rfc.md`，目標是建立一個包含 **1069 個** 分層抽樣人設（persona templates）的資料庫，代表臺灣約 2,300 萬人口結構。用途為 GEO（Generative Engine Optimization）Operator 在回應問題時載入對應的人設視角。

### 原始維度設計（RFC 定義）

| # | 維度 | 分層 |
|---|------|------|
| 1 | 年齡 | 8 層 |
| 2 | 性別 | 男 / 女 |
| 3 | 區域 | 6 大區域 |
| 4 | 教育 | 4 層 |
| 5 | 職業 | 11 類 |
| 6 | 個人收入 | 4 級 |
| 7 | 政治傾向 | 4 類 |
| 8 | 媒體習慣 | 5 類 |
| 9 | 家庭口數 | 5 級 |
| 10 | 家庭可支配所得 | 5 級 |
| 11 | 興趣嗜好 | ~16 種 |
| 12 | 婚姻狀況 | 4 類 |
| 13 | 戶籍地 | 22 縣市 |
| 14 | 物價分級 | 5 級（依居住地對應主計總處數據） |
| 15 | 家戶所得分級 | 5 級（依居住地對應主計總處數據） |

實際實作擴充為 **18 個維度**（含 background_story、political_stance、political_issues）。

## 3. 抽樣演算法

### 3.1 分層加權抽樣

```python
TARGET = 1069
# 96 個基礎組合（6 區域 × 8 年齡 × 2 性別），每個組合的目標人數
# 依 REGION_POP × AGE_DIST × SEX_DIST 加權
for region, rpct in REGION_POP:
    for age, apct in AGE_DIST:
        for sex, spct in SEX_DIST:
            n = max(1, round(rpct × apct × spct × TARGET))
```

### 3.2 四捨五入誤差調整

由於四捨五入導致總數 ≠ 1069，使用**比例殘差法**逐輪調整：
- 殘差正（不足）→ 找當前占比 / 目標占比差距最大的組合補 1
- 殘差負（超出）→ 找當前占比 / 目標占比差距最小的組合減 1

### 3.3 維度指派順序

每個 persona 依序獨立選取：

```
1. 教育 (EDU_PROB[age])
2. 職業 (OCC_PROB[age, sex])
3. 城市職業加權 (OCC_CITY_BOOST[residence])
4. 個人收入 (INC_PROB[occupation])
5. 政治傾向 (POL_PROB[region]; 未成年跳過)
6. 媒體習慣 (MEDIA_PROB[age])
7. 家庭口數 (FAMILY_SIZE_PROB[age])
8. 家庭所得 (FAMILY_INCOME_PROB[income, occupation])
9. 興趣 (HOBBY_POOL[age, sex])
10. 婚姻 (MARRIAGE_PROB[age, sex])
11. 戶籍地 (random.choice(REGION_TO_CITIES[region]))
12. 物價分級 (PRICE_TIER[residence])
13. 家戶所得分級 (HH_INCOME_TIER[residence])
14. 所得分級調整 (adjust_family_income_by_tier)
```

### 3.4 背景故事推論引擎

~30 條家庭狀況規則，依 (age, sex, marriage, occupation, family_size, family_income, region, income, price_tier) 選取適當的短語組合：

- **家庭組成短語**：11 種家庭型態 × 年齡/婚姻/性別調整
- **經濟狀況短語**：依物價分級 + 個人收入雙重加權
- **人生階段短語**：依年齡 + 職業組合
- **政治立場補強**：60+ 組 (政治傾向×職業群組) 交叉短語

### 3.5 Prompt Prefix 三模板

隨機指派三種模板之一（seed=42 確保可重現）：

| 模板 | 佔比 | 風格 | 範例 |
|:----|:---:|:----|:-----|
| A | 40% | 第三人稱敘述 | 「萌萌是國小女生，住在台北市。跟爸爸媽媽和一個弟弟住。平常喜歡繪畫、球類。」 |
| B | 35% | 第一人稱自然 | 「我叫萌萌，是國小女生，住台北市。跟爸爸媽媽和一個弟弟住。放假喜歡繪畫、球類。」 |
| C | 25% | 簡潔自然 | 「我是萌萌，國小女生，住在台北市，還在讀書。跟爸爸媽媽和一個弟弟住。興趣是繪畫、球類。」 |

### 3.6 經濟情境句（收入 vs 物價對比）

```python
# 核心邏輯範例
if income == ">8萬" and price_tier == "高":
    → "在台北市收入不錯，但物價高還是要算著花。"
elif income == "3-8萬" and price_tier == "極低":
    → "在南投縣這個收入還不錯，不用太省。"
elif income == "<3萬" and price_tier == "高":
    → "在台北市收入低，生活壓力很大。"
```

同一收入在不一樣城市產生完全不一樣的經濟感受。

## 4. 連貫性驗證與修正

在維度指派後，對矛盾組合進行自動修正：

| 規則 | 觸發條件 | 修正方式 |
|:----|:---------|:---------|
| 家庭收入偏低 | 5人家庭 + 收入 <3萬 | 提升至合理區間 |
| 未成年已婚 | 0-18歲 + 已婚 | 改為未婚 |
| 學生離婚 | 19-24學生 + 離婚 | 改為未婚 |
| 青壯年鰥寡 | 19-34歲 + 鰥寡 | 70%改為離婚/未婚 |
| 專業低學歷 | 35+科技/金融/醫療 + 國中以下 | 提升至大學 |
| 故事與家庭矛盾 | 家庭>1人 + 故事說「一個人過日子」 | 替換為適合短語 |
| 含飴弄孫 + 低收入 | 含飴弄孫 + 收入 <3萬 + 高物價 | 替換為「簡單過日子」 |
| 沒有生小孩 + 小孩開銷 | 故事同時出現矛盾短語 | 替換開銷短語 |

### QA 驗證規則（20 條）

產生後以 `qa_validate.py` 執行 20 條自動化檢查（R1-R20），全部通過才算合格。

## 6. 輸出格式

```json
{
  "id": "TW-P-0001",
  "name": "萌萌",
  "type": "虛構人設",
  "note": "人口加權抽樣（14維度，含背景推論+政治補強）",
  "dimensions": {
    "age": "0-11",
    "sex": "女",
    "region": "都會區(六都)",
    "education": "國中以下",
    "occupation": "學生",
    "income": "無收入",
    "politics": "無政治傾向",
    "media_diet": "家長主導",
    "family_size": "4",
    "family_income": "3-7萬",
    "hobby": ["繪畫", "球類"],
    "marriage": "未婚",
    "political_stance": "",
    "political_issues": [],
    "background_story": "跟爸爸媽媽和一個弟弟住。最近物價漲得很有感，日子更緊了。每天放學就是寫作業和看卡通。住在都會區。",
    "residence": "台北市",
    "price_tier": "高",
    "hh_income_tier": "極高"
  },
  "reference_pre_prompt": "【萌萌】0-11/都會區(六都)/女/未婚/學生/國中以下/4人/3-7萬/繪畫、球類",
  "prompt_prefix": "萌萌是國小女生，住在台北市。跟爸爸媽媽和一個弟弟住。平常喜歡繪畫、球類。"
}
```

每筆包含：
- **`dimensions`**：18 個維度的結構化數據
- **`reference_pre_prompt`**：簡潔摘要，供 GEO Operator 過濾
- **`prompt_prefix`**：自然語言人設描述，可直接作為 LLM 的系統提示

## 8. 再現性

相同的程式 + seed=42 應產生完全一致的結果：

```bash
git checkout <commit-hash>
cd hermesa3/persona
python3 _generate_997.py    # → tw_persona_1069.json
python3 qa_validate.py      # → 20 rules, all passed
```

由於 ODRP014 API 為動態資料，不同日期擷取的回傳總數可能微幅影響校正結果，但整體分布差異應在 ±1% 以內。

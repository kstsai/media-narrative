# Persona 人設分析法 — 應用於選舉民調與媒體風向

## 概述

從創作領域的「角色人設」與 UX 領域的「用戶畫像」出發，探討能否用 Persona 方法論取代或補強傳統民調，進行選舉勝算預測、選民心理建模、候選人品牌定位分析。

## 異想一：選舉 = 角色養成遊戲

將候選人視為可抽角色卡，每張卡有屬性面板（ATK/DEF/SPD）與技能。

選民 = 玩家，各有不同隊伍配置需求。
模擬目標：在當前的 meta 環境中，誰的 kit 最配主流選民組成？

**範例（2026 竹北市長）：**
- 吳旭智 [★★★ 藍] — 技能：地方組織Buff / 藍白合不確定性
- 邱臣遠 [★★★ 白] — 技能：空戰媒體聲量 / 地方組織薄弱
- 綠營人選 [TBD] — 技能：鄭朝方政績繼承 / 執政黨包袱

## 異想二：候選人「人設化學反應」矩陣

非零和競爭，而是 1+1 > 2 或 < 2 的化學反應：

- **A vs B（一對一）：** 勝負取決於人設對比鮮明度。務實建設派 vs 空戰明星型 → 在不同選民畫像中各有勝場。
- **A vs B vs C（三方混戰）：** 關鍵不是誰第一，而是票的流動方向。人設過於接近的候選人會互相削弱（例：藍白在年輕工程師選民中人設重疊 → 互相分票 → 綠營漁翁得利）。

## 異想三：「人設不匹配度」指數（Anti-Persona）

傳統民調問「你支持誰」（正面問題）。反過來問：「每個候選人最讓哪一群選民反感？」

找出每個候選人的人設紅線，估算這些紅線涵蓋多少選民：

- 吳旭智 → 非藍選民覺得「又是一個地方派系」
- 邱臣遠 → 在地選民覺得「空降部隊」
- 綠營人選 → 藍白選民覺得「執政黨包袱」

## 異想四：選舉旅程地圖（Election Journey Map）

套用 UX 的 Customer Journey Map 到選舉決策：

1. **無感期**（選前一年）→ 不在意
2. **察覺期**（半年）→ 開始聽到名字
3. **印象形成期**（三個月）→ 建立人設標籤
4. **比較期**（一個月）→ vs 對手
5. **決策期**（投票前兩週）→ 誰的故事更打動我

每個階段，選民在意的「人設要素」不同。選戰策略可依此在不同階段投放不同的人設訊息。

## 異想五：情感共鳴力評分

用 NLP 分析候選人公開言論，不只是看 policy，而是看：

- **敘事類型：** 英雄旅程 / 受害者敘事 / 改革者敘事
- **情感色調：** 自信 / 焦慮 / 憤怒 / 溫暖
- **情感鏡像：** 選民在候選人身上看到自己想成為的樣子嗎？

然後對各選民 persona 做情感共鳴配對，預測感染力。

## 異想六：人設風向球 — 即時監控系統

將上述全部自動化：

1. 每日爬候選人新聞、FB、造勢影片逐字稿
2. 自動更新「人設面板」（外在標籤、內在特質、角色弧光）
3. 若人設突然偏移（務實派 → 打泥巴戰）→ 人設崩壞事件 → 自動推估票的流失方向

這不是民調，但比民調更早看到趨勢轉折。

## 應用案例構想

### 竹北市長選舉（2026）

情境模擬：
- A vs B（一對一）：自由心證
- A vs B vs C（三方）：分票效應
- 影響因子：藍白合進度、鄭朝方接班人選、竹科選民投票率

### 擴充方向

- 加入候選人過往選舉得票 data point 校準模型
- 加入媒體報導量（見 hermesa3/mediawatch/media-narrative-prototype.md）
- 加入 PTT/Dcard 社群聲量作為「人設共鳴度」proxy

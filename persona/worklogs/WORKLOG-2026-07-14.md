# Persona 模擬問卷方法論 — Spawn Agent ①自我 vs ②遭遇
**日期**: 2026-07-14（三）08:00-08:30 CST (UTC+8)
**費率**: NT$2,000/hr
**內容**: 方法論驗證 — Persona 篩選 + Spawn Agent 2×2 完整實驗

## 工作內容

### — Persona 篩選（康是美展店問卷）
- 從 `tw_persona_1069.json` 篩選適合康是美（藥妝）展店問卷的 3 個 persona
- 篩選條件：女性、25-54 歲、有逛街購物興趣、地理涵蓋北/竹/南
- 選出：
  - **麗華 (TW-P-0468)** — 47歲, 台北市, 科技業, 3-8萬, 空巢期雙薪
  - **靜怡 (TW-P-0797)** — 37歲, 新竹市, 科技業, 3-8萬, 家有學齡子女
  - **淑貞 (TW-P-0349)** — 41歲, 台南市, 家管, 3-8萬, 注重養生

### — 同一 Agent 模擬（對照組）
- 以同一 Agent 分別回應麗華、淑貞的「數位帳戶」問卷
- 結果：三人推薦品牌相似（將來銀行為主），有 contamination bias

### — Spawn Agent 2×2 完整實驗
- 設計 2 種模擬模式 × 2 個問題 × 3 個 persona = 12 組模擬
- 使用 `delegate_task` 派出獨立 sub-agent（deepseek-v4-pro），每批 3 個並行

#### 模式定義
| 模式 | 說明 | Sub-agent 角色 |
|:---:|:------|:--------------|
| **① 自我模擬** | Sub-agent「是」Persona，回答問卷 | 扮演麗華/靜怡/淑貞本人 |
| **② 遭遇模擬** | Sub-agent「遇到」Persona來問，回答她 | 扮演專家/朋友，針對 Persona 背景給建議 |

#### 關鍵發現
1. **①自我 vs ②遭遇的本質差異**：自我模擬像填問卷（聚焦個人需求）；遭遇模擬像專家給建議（分析背景、客製化、甚至反駁偏好）
2. **跨區域展店策略對應清晰**：台北→通勤簡潔店、新竹→母嬰社區店、台南→健康調理店
3. **偏好決策**：正式問卷模擬使用 spawn sub-agent 方式，確保 bias-free

- 偏好決策：正式問卷模擬使用 spawn sub-agent 方式
